The apps

For this challange we chose the following apps:

  • Discord

    Discord is a VoIP and instant messaging social platform. Users have the ability to communicate with voice calls, video calls, text messaging, media and files in private chats or as part of communities called "servers".

  • Messenger

    Messenger is a proprietary instant messaging app and platform developed by Meta Platforms. Originally developed as Facebook Chat in 2008, the company revamped its messaging service in 2010.

Affordances, visual variables and interaction flows

Affordances

The hints that the product/device offers in order to help us realise how it should be used: in a layout we have buttons with labels.
An example for our day to day life is the following: the existence of a doorknob tells us that that object can be opened

  • Discord

    The first experience offers some hints. The first thing we see is the option to join a public community (server) or to create your own.

    The landing page's main function is to search a friend in case you want to start communicating with another user.

    In case the user is confused, in the top right corner there is a "help" button that redirects to a FAQ page with more information and the option to search for the problem you met.

  • Messenger

    For a new user, the interface is completely empty and it's pretty difficult to know what to do.

    For a user that already has a few friends, the main functions (chat, calls) are intuitive and clear

Visual variables

A visual variable is an aspect of a graphical object that can visually differentiate it from other objects, and can be controlled during the design process.
Those variables consist of: position, size, shape, colour, orientation and texture.

  • Discord

    Usually, all the visual elements are well differentiated, organised in clear and persistent sections.
    For example:

    Servers are always round icons placed on the left side of the screen in a vertical list that has a dark shade of grey.

    Then there is another vertical list that has a lighter shade of grey where:

        When we are on a server's window, we can observe communication channels, graphically identified as voice/text channels accompanied by their name.
        When we are on the direct messages window, we can observe the friend's list made form the user's profile picture and their name.
        Excepting the vertical orientation and the position on the screen, those differences are easy to spot.

    After this we have the main area that represents the chat room we have selected.
    Besides those there are other sections that are sepparated just as well.

  • Messenger

    On the mobile app, we only have acces to a single view, therefore there no ambiguity is present between sections.
    On the desktop/web app we have, just like Discord, 3 main sections.
    This time they are not differentiated trough colour, but trough borders.

    First we have a main menu from which we can acces different chat categories that are illustrated trough icons:

        - Normal chats
        - Chats with currently
        - Marketplace chats
        - Chat requests
        - Archives

    It's fairly easy to determine in what section you are since the title of the next section changes accordingly.
    Then we have the chat section: a vertical list with every conversation, where we have the user's profile pic, name and the last messege.
    Then we have the main section: the chat.

Interaction flows

An interaction flow refers to the sequence of actions or steps that a user takes to accomplish a specific task or goal.

  • Discord

    Usually, any flow is intuitive and it's not formed from too many steps (around 5 steps for anything).

    An example would be when an user receives a notification in a text channel. First the task bar icon receives a pip, then, if the messege is from a server, a white pip appears next to the server, and the channel where the messege is sent has a bold font and white colour.

    Also, the user cna use the inbox to see unread messeges.

  • Messenger

    Any flow that is related to communication is intuitive and simple. dar anything that has to do with the account(changing a password) depends on Facebook, which requires additional steps.

    We will follow a simillar flow for messenger. When an user receives a message, they get a pip in the taskbar. In the contacts list(the one that sent the message appears at the top of the list) the username is highlighted in bold.

Visual metaphors, idioms and text-only interaction

Visual metaphors and idioms

There are multiple simillarities between the two apps.

They both are sepparated in 3 sections.

The icons they use are generic (a phone for voicalls, a videocamera for video calls, a magnifying glass to search a message, etc.).

An exception is that on the web/desktop apps Messenger has a button to send a message while Discord doesn't have this button.

Text-only interaction

We will take into consideration the sections describet previously and the base functionalities for this excercise.

  • Discord

    1. The server, channels and friends list can be represented trough a structure simillar to documents on a Windows command prompt.

      ls servers
      cd name_server

      ls name_server
      cd name_channel
      read messages
      write message

      ls friends
      cd name_friend
      read messages
      write message

    2. During a call we have the following options: mute, deafen, share screen, close call.
      If you're on a server you have to acess a voice channel.

      On the command line, after entering a conversation with another user, we can use the following commands:

      start video_call
      start voice_call
      join_call
      reject_call
      exit_call
      deafen me
      mute me
      mute user

      For servers, from the channels list we can chose a voice channel where we can use the following commands:

      join channel
      exit channel
      mute me
      mute user
      deafen me

  • Messenger

    Being a simillar app, we will apply the same implementation for a command line app.

    An exception is that Messenger doesn't have servers, but it has groups. But those groups are more simillar to a normal conversation.

Cognitive overload

  • Discord

    An example for a cognitive overload is that there can be servers with a big channel number that aren't grouped, which can make finding the right channel difficult.

    Also, if there are a lot of users in a voice channel, communication is difficult.

  • Messenger

    If there are large groups. there is the posibility to have a big messages flow that the user can'tkeep up with.

Three improvements

  • Discord

    • Platforms: Web/Desktop
      Problem: There is no send button, the message has to be sent by pressing Enter.
      Solution: Adding a send button.

    • Platforms: Web/Desktop/Mobile
      Problem: There is no on-boarding for activities and the library.
      Solution: Addition of on-boarding pop ups and tips for new users.

    • Platforms: Web/Desktop
      Problem: The abillity to share your sound with full screen isn't there, but sharing the sound of only one app is possible.
      Solution: Making it so sound is shared even when the whole screen is shared.

  • Messenger

    • Platforms: Web/Desktop/Mobile
      Problem: Big conversation number. Even though some can be archived if they are of no interest, there is the posibility to have more active conversations.
      Solution: The ability to group conversations in customizable folders.

    • Platforms: Web/Desktop/Mobile
      Problem: The apps dependence on Facebook.
      Solution: Adding the necessary views to mnaage your account.

    • Platforms: Web/Desktop/Mobil
      Problem: We may need to find a certain message often, which can be difficult with the passing of time.
      Solution: The ability to pin a message or to add it to favourites.

Empiric laws

Fitts'

Fitts' law states that the amount of time required for a person to move a pointer to a target area is a function of the distance to the target divided by the size of the target.Thus, the longer the distance and the smaller the target’s size, the longer it takes.

  • Discord

    Overall, Discord's elements are small and distant.
    Example: to call a friend, we must open the conversation and then press on the call icon.Those buttons are sitting on opposite ends of the window.

    The menus are positioned on the edge of the window on the Web/Desktop version.
    On the mobile version we need to swipe, then press on the profile icon from where we can access settings.

  • Messenger

    Messenger has a very similar case to discord, where the buttons are sitting on opposite sides and the icons are fairly small, but they are a bit bigger than Discord's.

Hick's

Hick's law describes the time it takes for a person to make a decision as a result of the possible choices: increasing the number of choices will increase the decision time.

  • Discord

    Discord's settings are placed in a single view. Those are sepaprated in multiple categories(User settings, Billing settings, etc.), that also have subdivisions, from which you access multiple settings.

    Even if they are well divided, there are a few ambiguities between categories and they are visually differentiated.

  • Messenger

    Discord's settings are placed in a single view. Those are sepaprated in multiple categories(User settings, Billing settings, etc.), that also have subdivisions, from which you access multiple settings.

    Even if they are well divided, there are a few ambiguities between categories and they are visually differentiated.

Steering

The steering law is a predictive model of human movement that describes the time required to navigate, or steer, through a 2-dimensional tunnel. The tunnel can be thought of as a path or trajectory on a plane that has an associated thickness or width, where the width can vary along the tunnel. The goal of a steering task is to navigate from one end of the tunnel to the other as quickly as possible, without touching the boundaries of the tunnel.

Both apps have short tunnels. Approximately 4 or 5 sections.
